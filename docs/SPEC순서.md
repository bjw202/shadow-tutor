🔍 단계별 상세 설명

1단계: 프로젝트 초기 설정

- Next.js 16 프로젝트 생성
- Tailwind CSS + shadcn/ui 설정
- Zustand 스토어 기본 구조
- 폴더 구조 생성

2단계: 텍스트 업로드 & 파싱 ← 핵심 데이터 흐름

- 파일 업로드 (Dropzone)
- 텍스트 직접 입력 (Textarea)
- MD/TXT 파싱
- 문장 단위 자동 분리 (NLP 기반)
- 세그먼트 목록 생성 & 저장

3단계: TTS 재생 (기본)

- OpenAI TTS API 연동 (/api/tts 라우트)
- 단일 문장 재생
- 오디오 버퍼 관리
- 에러 핸들링

4단계: 플레이어 UI & 하이라이트

- 오디오 플레이어 컴포넌트
- 재생/일시정지 버튼
- 현재 문장 하이라이트 표시
- 이전/다음 문장 네비게이션
- 전체 문장 수 표시

5단계: 학습 모드

- 연속 모드: 전체 텍스트 순차 재생
- 쉐도잉 모드: 문장 재생 → 대기 → 자동 진행
- 모드 전환 UI
- 자동 진행 로직

6단계: 설정 옵션

- 읽기 속도 (0.8x, 1.0x, 1.2x)
- 대기 시간 (3초, 5초, 7초, 10초)
- 설정 저장 (Zustand persist)

7단계: 진행률 관리

- 현재 위치 LocalStorage 저장
- 앱 재접속 시 이어서 학습
- 진행률 바 표시
- 세션 관리

8단계: PWA & 오프라인

- Service Worker 설정
- 오디오 캐싱
- 오프라인 재생
- 홈 화면 설치

---

🎯 MVP 범위 제안

Phase 1 MVP (핵심 기능): 1-5단계

- 텍스트 업로드 → TTS 재생 → 쉐도잉 모드

Phase 2 (완성도): 6-7단계

- 설정 & 진행률 저장

Phase 3 (배포): 8단계

- PWA 오프라인 지원
